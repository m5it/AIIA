When you run `model.save_pretrained('path/to/model/directory')`, it saves the following files:

*   `config.json`: The model's configuration file.
*   `pytorch_model.bin`: The PyTorch model weights (in binary format).
*   `vocab.txt`: The vocabulary file, which contains the wordpiece tokens used by BERT.

However, for a complete and consistent model archive, it's a good practice to copy the following files as well:

*   `tokenizer.json`: This file contains the tokenizer configuration, including tokenization rules and special token definitions. It's essential 
for reproducing the exact behavior of your model.
*   `tokenizer_config.json`: Although this is not explicitly mentioned in the Hugging Face Transformers documentation, some models (like BERT) 
include a `tokenizer_config.json` file to store additional tokenizer configurations.

The `model.safetensors` file is an optional file that's created when you use the `save_pretrained()` method with the `safetensors` format. It 
stores the model weights in a more compact and efficient binary format, suitable for large models or low-memory environments.

To ensure your model archive is complete and consistent:

1.  Copy all files from the original `transformers` package (e.g., `bert-base-uncased`) to your custom model directory.
2.  Verify that the following files are present:
    *   `config.json`
    *   `vocab.txt`
    *   `tokenizer.json`
    *   `tokenizer_config.json`

If you're using a specific model or configuration, make sure to check its original documentation for any additional required files.

Here's an example of how to copy the necessary files:

```python
import os

# Define your custom model directory and the source package directory
model_dir = 'path/to/model/directory'
source_pkg_dir = 'path/to/transformers/package'

# Copy the required files from the source package
files_to_copy = [
    'config.json',
    'vocab.txt',
    'tokenizer.json',
    'tokenizer_config.json'  # Some models might have this file; check the original documentation.
]

for file in files_to_copy:
    src_file_path = os.path.join(source_pkg_dir, file)
    dst_file_path = os.path.join(model_dir, file)

    if os.path.exists(src_file_path):
        print(f"Copying {file} from source package to custom model directory.")
        with open(dst_file_path, 'wb') as f_dst:
            with open(src_file_path, 'rb') as f_src:
                f_dst.write(f_src.read())
    else:
        print(f"Skipping non-existent file: {file}")

```

This script copies the necessary files from the source package to your custom model directory. Make sure to adjust the `source_pkg_dir` variable 
according to your specific requirements.

By copying these additional files, you'll ensure that your custom model archive is consistent and easy to reproduce.

>>> what is different between created binary with model.save_pretrained or torch.save(model.state_dict(),'yourmodel.bin') ?
When you use `model.save_pretrained()` to save a Hugging Face Transformers model, it saves the following files:

*   `config.json`: The model's configuration file.
*   `vocab.txt`: The vocabulary file, which contains the wordpiece tokens used by BERT.
*   `tokenizer.json`: The tokenizer configuration file (which we discussed earlier).
*   `model.safetensors` (optional): A binary file containing the model weights in a compressed format.

On the other hand, when you use `torch.save(model.state_dict(), 'yourmodel.bin')`, it saves only the model's state dictionary to a single binary 
file (`'yourmodel.bin'`). This is equivalent to saving the model's weights, but not its configuration or tokenizer information.

Here are some key differences between the two approaches:

1.  **Configuration and tokenizer**: `save_pretrained()` saves the entire model archive, including the configuration and tokenizer files. In 
contrast, `torch.save(model.state_dict(), ...)` only saves the state dictionary of the model.
2.  **Binary format**: The `model.safetensors` file created by `save_pretrained()` is a binary file that uses the SAFETensors format to store 
the model weights in a compressed and efficient manner. In contrast, `torch.save(model.state_dict(), ...)` saves the state dictionary as a 
regular PyTorch binary file.
3.  **Model loading**: When you load a saved model using `model = AutoModel.from_pretrained('yourmodel')`, it automatically loads both the 
configuration and tokenizer files from the archive, in addition to the weights. In contrast, if you use `torch.load('yourmodel.bin', 
map_location=device)`, you'll only get the state dictionary of the model.
4.  **Reusability**: Saved models using `save_pretrained()` are more reusable because they include all necessary metadata (configuration and 
tokenizer). This makes it easier to load and fine-tune a saved model.

Choose `model.save_pretrained()` when:

*   You need to save your model's configuration, tokenizer, and weights together.
*   You want to make your model easily reproducible and shareable with others.
*   You plan to use the model in various tasks or workflows that require access to its metadata.

Choose `torch.save(model.state_dict(), ...)` when:

*   You only need to save the model's weights and don't care about preserving the configuration or tokenizer information.
*   You're working within a specific PyTorch project and want to save the model state dictionary in a standard format.
