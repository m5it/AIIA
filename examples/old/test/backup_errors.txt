The error message indicates that the `encode_plus` method of the `tokenizer` object is trying to encode the label values (which are integers) as 
input IDs.

When you call `labels.apply(lambda x: tokenizer.encode_plus(x, ...)`, it's applying the lambda function to each value in the `labels` series. 
Since the labels are integers, they can't be encoded directly by the `encode_plus` method.

To fix this issue, you should only apply the encoding to the text inputs, not the labels. You should use the integer label values as is when 
creating your dataset and data loaders.

Here's an updated code snippet:

```python
inputs = df['text']
labels = df['label']

tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')

inputs_tok = tokenizer.encode_plus(
    inputs,
    add_special_tokens=True,
    max_length=512,
    padding='max_length',
    truncation=True,
    return_attention_mask=True,
    return_tensors='pt'
)

class BERTDataset(torch.utils.data.Dataset):
    def __init__(self, inputs_tok, labels):
        self.inputs_tok = inputs_tok
        self.labels = labels

    def __getitem__(self, idx):
        input_ids      = self.inputs_tok['input_ids'].iloc[idx]
        attention_mask = self.inputs_tok['attention_mask'].iloc[idx]
        label          = torch.tensor(self.labels.iloc[idx])

        return {
            'input_ids': input_ids,
            'attention_mask': attention_mask,
            'labels': label
        }

    def __len__(self):
        return len(self.labels)
```

In this updated code, the `encode_plus` method is only applied to the text inputs, and the integer label values are converted to PyTorch tensors 
using `torch.tensor`.
